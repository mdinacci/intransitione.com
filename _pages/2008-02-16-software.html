PicInMotion
[button link="#"]Java[/button]  [button link="#"]Android[/button]

UPDATE: source code available on: github.

My first complete Android application. PIM is a motion detector. When it notices a moment in the camera preview it takes a picture and store it on the sd card. The app is not available on the market as it's not finished and I have no intention to market it, there are already other free and paying application that does a much better job available on the market. I made this app just for learning purposes so it's not as polished as a proper application. I've also written an article to explain how the motion detection works. Download it or use the QR code below to download it directly on your phone.
To install it on the phone, either scan the QR code with a barcode scanner or download the apk on your disk and copy it on the phone. Then use whatever software you like that installs applications from apk files. Two examples are App Installer and APK install.

The images are stored at /sdcard/Pictures/pim, just use a file manager to see and delete the images.

I only tested it on a Motorola Milestone (first model, Android 2.1), as I'm not selling it I didn't test it throughly so it's possible it's not going to work on your device. Use at your risk.

Photoflow

[button link="#"]Python[/button]  [button link="#"]Qt (PySide)[/button]
A small python utility to copy new images and videos from a camera (more in general from any location, I just use it for the camera) to disk. It remembers the last image you took so if you go out to take some new pictures the script will only copy the new ones. It also organizes the images using a year/month/day structure by reading the date inside the EXIF tags. Images and videos can be stored in different folders. Now comes with a GUI, in QT, also for Windows. It's a big download (11MB) since it packages all the requirements, Python included.

Command-line usage: 

photoflow.py -i /source/photos /dest/photos -v /source/videos  /dest/videos

For the GUI just run: 

python gui.py
on Linux and double-click 
gui.exe
on Windows.

Download source from photoflow.py from GitHub

ID3Tree
[button link="#"]Python[/button]

Another small utility, it reads MP3 tags inside the music files and create a directory structure like 

artist/album/song_title.mp3
moving the files to the correct destination. It needs only author, album and title tags. If one of them are not found they are substituted with Unknown Album and Unknown Artist. Be careful you may overwrite some files if you have many untagged files.

Usage: 

fix_music.py /path/to/music /path/to/destination

Download it. [/box]

Synkie
[button link="#"]Python[/button]  [button link="#"]GTK[/button]

An ambitious attempt to create an application that could synchronize data especially between my phone and my computer. It's still at very alpha stage, I'm going to write an article as soon as the code is more stable and setup a public repository for the code.


At the moment it only does a diff between two directories and copy the files existing in the source folder to the destination folder. It also removes files on the destination folder when not selected on the source folder. It requires a directory in ~/.config/synkie with sources.db and destinations.db installed. I'm going to write an article soon to explain how to create these files.

Source code available from github.

Download it.
